# 🕹️ Pygame Essential Checklist (Minimum Viable Mastery)

## Chapter 1 — Window + Game Loop (CORE)

This is the foundation of everything.

- [ ] Initialize Pygame
- [ ] Create window (`display.set_mode`)
- [ ] Set window title
- [ ] Understand the game loop:
  - [ ] event loop
  - [ ] update step
  - [ ] render step

- [ ] Control FPS (`Clock.tick`)
- [ ] Quit cleanly (`pygame.quit`)

**Goal:** You can create a stable running window at 60 FPS.

---

## Chapter 2 — Input Handling (VERY IMPORTANT)

This is how games become interactive.

- [ ] Keyboard input (keydown / keyup)
- [ ] Continuous key state (`get_pressed`)
- [ ] Mouse position
- [ ] Mouse click detection
- [ ] Event types (QUIT, KEYDOWN, MOUSEBUTTONDOWN)

**Goal:** You can control a snake or object reliably.

---

## Chapter 3 — Drawing Basics (RENDERING CORE)

No fancy graphics needed.

- [ ] Draw rectangle
- [ ] Draw circle
- [ ] Draw lines
- [ ] Fill screen with color
- [ ] RGB color system
- [ ] Screen coordinate system (0,0 top-left)

**Goal:** You can visually represent game state.

---

## Chapter 4 — Game State Model (MOST IMPORTANT FOR YOU)

This is where most beginners fail.

- [ ] Separate **game state vs rendering**
- [ ] Store positions as data (not visuals)
- [ ] Update logic BEFORE drawing
- [ ] Represent entities as objects or structs:
  - snake
  - food
  - obstacles

**Goal:** Game logic is independent of Pygame.

---

## Chapter 5 — Time & Movement

Critical for simulations.

- [ ] Delta time concept (basic understanding)
- [ ] Frame-based movement
- [ ] Grid-based movement (VERY important for Snake)
- [ ] Movement speed control

**Goal:** Movement is predictable and stable.

---

## Chapter 6 — Collision Detection (ESSENTIAL)

Used everywhere in games + simulations + backend modeling mindset.

- [ ] Rectangle collision (`colliderect`)
- [ ] Circle collision (distance-based)
- [ ] Point vs grid collision
- [ ] Self-collision logic (Snake body)

**Goal:** You can detect interactions between objects.

---

## Chapter 7 — Basic Structuring (IMPORTANT FOR YOUR ROADMAP)

This is where Pygame becomes “engineering”.

- [ ] Game class wrapper
- [ ] Separation:
  - [ ] input handler
  - [ ] game logic
  - [ ] renderer

- [ ] Avoid global variables
- [ ] Central game state object

**Goal:** Your code looks like a system, not a script.

---

## Chapter 8 — Simple Assets (OPTIONAL MINIMUM)

Only what you actually need.

- [ ] Load images
- [ ] Scale images
- [ ] Basic sprite concept (optional)
- [ ] Draw text (score, UI)

---

## Chapter 9 — Debugging Tools (VERY USEFUL)

- [ ] Print debugging state
- [ ] Draw debug shapes (hitboxes, positions)
- [ ] Visualizing grid
- [ ] FPS display

---

## Chapter 10 — Event-Driven Thinking (IMPORTANT CONCEPT)

Not heavy architecture, just awareness:

- [ ] Events vs continuous update
- [ ] Input → state change → render cycle
- [ ] Avoid mixing logic inside rendering

---

# 🚫 What You SHOULD NOT Learn (for your goal)

Skip completely:

- physics engines
- shaders
- 3D graphics
- sprite animation systems
- complex audio systems
- tile engines
- Pygame “framework building”

These will distract you from backend + AI goals.

WITHOUT looking up tutorials.
